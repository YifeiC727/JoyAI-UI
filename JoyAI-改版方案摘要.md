# JoyAI 改版方案摘要

---

## 核心思路

本次改版围绕**信息架构重构 + 首页体验升级**两条主线，将 JoyAI 从一个"聊天工具"升级为"AI 能力展厅与创作平台"。

---

## 主要改动

### 1. 入场动画（Splash Intro）

用户首次进入时展示约 2.5 秒的品牌入场动画：

- 深紫渐变全屏背景 + 微粒飘动
- JoyAI Logo 从中心弹出，带光环脉冲扩散
- 品牌名"JoyAI"与 tagline"探索无限可能"依次淡入
- 整体向上滑出淡去，露出主页内容

### 2. 首页 Hero 区重构

**改版前**：直接展示"JoyAI 能力平台"标题 + 四个小卡片

**改版后**：

- **大标题**："灵感**即刻实现** / 体验 · 交互 · 创作"（渐变紫色强调）
- **副标题**："用 JoyAI 探索未来的无限可能"
- **右上 3D 装饰**：CSS 玻璃方块 + 气泡 + 闪烁光点，带浮动动画
- **统一输入框**：用户可直接在首页输入指令，选择模型后跳转到对应功能并新建对话

### 3. 首页模型选择器（下拉式）

首页输入框下方设置下拉模型选择器，可选模型：

| 模型 | 跳转目标 |
|------|---------|
| JoyAI-Flash | 文字助手对话 |
| JoyAI-VL | 视觉理解对话 |
| JoyAI-Image | 视觉创作（快速创作） |
| 视频生成 | 视觉创作（Echo Studio） |

选择模型 → 输入指令 → 按回车 → 自动跳转到对应 Tab 并新建对话发送消息。

### 4. 能力展示卡片升级

**改版前**：四个小白卡（文字助手 / 视觉理解 / 视觉创作 / 语音能力），列出功能清单

**改版后**：四张高级感卡片，展示产品级能力入口：

| 卡片 | 功能 | 点击跳转 |
|------|------|---------|
| 实时识别 | 摄像头实时画面理解与对话 | 视觉理解 → Live Vision Pro |
| Echo Studio | AI 绘图、AI 视频、长视频创作 | 视觉创作 → Echo Studio |
| World Model | 实时可交互的开放世界 | 视觉创作（预留） |
| JoyAI-Sim | 物理仿真与数字孪生 | 视觉创作（预留） |

卡片样式：磨砂玻璃底 + hover 光泽扫过动效 + 右上角模糊色彩光晕 + SVG 图标 + 各自主题色"立即体验"链接

标题从"强大能力，触手可及"改为"**探索 AI，体验未来**"

### 5. 侧边栏导航结构

保持四大能力域一级导航不变：

```
JoyAI（点击回首页）

能力区域
├── 文字助手
├── 视觉理解
├── 视觉创作
└── 语音能力

最近使用
├── 周报总结助手 [文字]
├── 竞品 UI 截图分析 [视觉]
└── 京东 IP 吉祥物 [创作]

用户信息
```

各能力域内部设有 Tab 切换：
- **视觉理解**：聊天模式 / Live Vision
- **视觉创作**：快速创作 / Echo Studio
- **语音能力**：Talker 语音聊天 / 音色创作 / AI 配音

每个功能页面右上角提供「+ 新对话」按钮。

### 6. 隐藏能力提升为正式入口

| 能力 | 改版前位置 | 改版后位置 |
|------|-----------|-----------|
| 实时识别 | 视觉解读页面内的小圆点 | 首页能力卡片 + 视觉理解 Tab |
| Echo Studio | 视觉创作页面内的小圆点 | 首页能力卡片 + 视觉创作 Tab |
| World Model | 不存在 | 首页能力卡片（预留） |
| JoyAI-Sim | 不存在 | 首页能力卡片（预留） |

## 模型矩阵与导航架构

### 完整模型清单

| 模型名称 | 能力 | 所属团队 |
|---------|------|---------|
| JoyAI-Flash | 文字对话、创作、翻译、编程 | 文字 |
| JoyAI-VL | 图片/视频/文档理解（1个模型，多种输入） | 视觉理解 |
| JoyAI-VL-Interact | 实时视觉理解（直播/实时视频流） | 视觉理解 |
| JoyAI-Image | 图片生成 | 图片创作 |
| JoyAI-Image Edit | 图片编辑 | 图片创作 |
| JoyAI-Video | 视频生成 | 视频创作 |
| JoyAI-Video Edit | 视频编辑 | 视频创作 |
| JoyAI-Talker | 语音聊天 | 语音 |
| JoyAI-Voice（音色创作） | 自定义音色 | 语音 |
| JoyAI-Voice（多角色配音） | 多角色 AI 配音 | 语音 |

### 核心矛盾

VL 是 **1个模型处理多种输入**（图片/视频/文档/实时），而视觉创作是 **4个独立模型**（Image / Image Edit / Video / Video Edit）。如果按原来的4个一级分类，视觉创作要塞4个模型，其他只有1-2个，对各团队不够公平。

### 需设计的共识与要点

**各组共识：**
1. 首页需有展示所有模型的导入页
2. 交互页左侧一级分发，上侧二级分发

**需设计的点：**
1. 导入页的模型分布方式（待定）
2. 交互页一级及二级对应关系
3. 整体设计风格

### 推荐方案：5个一级（按模态拆创作）

将"视觉创作"拆为"图片创作"和"视频创作"，使每个团队拥有独立的一级入口：

```
L1 左侧一级        L2 顶部二级 Tab
─────────────       ──────────────────────────────────
文字助手             （无 Tab，直接对话）
                     模型：Flash

视觉理解             图片理解 │ 视频理解 │ 文档理解 │ 实时视觉
                     VL         VL         VL         VL-Interact

图片创作             图片生成 │ 图片编辑
                     Image      Image Edit

视频创作             视频生成 │ 视频编辑
                     Video      Video Edit

语音能力             语音聊天 │ 音色创作 │ AI 配音
                     Talker     Voice      Voice
```

**优点：**
- 每个 L1 下 2-4 个 Tab，分布均匀
- Image 团队和 Video 团队各有独立一级入口，公平展示
- VL 的多能力通过 4 个 Tab 充分展示（同一模型不同输入形式）
- 用户认知清晰：理解 ≠ 创作，图片 ≠ 视频

### 备选方案：4个一级（保持现有结构）

```
L1 左侧一级        L2 顶部二级 Tab
─────────────       ──────────────────────────────────
文字助手             （无 Tab）
                     Flash

视觉理解             图片理解 │ 视频理解 │ 文档理解 │ 实时视觉
                     VL         VL         VL         VL-Interact

视觉创作             图片生成 │ 图片编辑 │ 视频生成 │ 视频编辑
                     Image      Image Edit  Video      Video Edit

语音能力             语音聊天 │ 音色创作 │ AI 配音
                     Talker     Voice       Voice
```

**问题：** 视觉创作塞了 4 个 Tab 会拥挤，且 Image 和 Video 团队共享一个 L1 可能不够公平。

---

- 各功能的聊天界面、生成界面等**右侧主体区域不改动**
- 不涉及后端接口或数据模型的变化
- 视觉风格（配色、字体、间距等）为原型示意，非最终定稿
- World Model 和 JoyAI-Sim 为预留入口，暂无独立功能页面
