# JoyAI 首页卡片系统 — 设计与技术细节

---

## 一、布局方案

**扇形展开排列**，卡片从中心向两侧展开，中间正面朝前，两侧微旋转+缩小，类似手持扑克牌的视觉效果。

| 参数 | 值 |
|------|-----|
| 卡片尺寸 | 260×360px，圆角 24px |
| 卡片间距 | 200px（中心点间距） |
| 旋转角度 | 每偏移 1 位 ±6° |
| 缩放 | 每偏移 1 位 ×0.97 |
| Y 轴下沉 | 每偏移 1 位 +18px |
| z-index | 中心最高，两侧递减 |

## 二、三段式交互

### Phase 1 — 即时轻反馈（0ms）

- 鼠标进入卡片范围 → 边框变紫 + 阴影增强
- 其他卡片不动，整排不平移
- 目的：让用户知道"我在这张卡上"

### Phase 2 — 延迟居中（180ms 停留）

- 鼠标在同一张卡片上**真正停住** 180ms 后 → 整排平移让该卡片到正中央
- 选中卡片放大 1.05× + 上浮 14px
- 其他卡片缩小 0.98× + 降透明度到 0.75
- **关键防抖逻辑**：每次 mousemove 都重置计时器，只有鼠标真正停下才触发。避免"卡片移动后新卡片到鼠标下 → 连锁跳动"

### Phase 3 — 点击锁定

- 点击卡片 → 锁定居中状态，鼠标移开也不复位
- 再次点击同一张 → 进入该功能（或解锁）
- 点击其他卡片 → 切换锁定目标

## 三、防抖核心技术

### 问题根源

hover 触发卡片平移 → 新卡片跑到鼠标下 → 触发新 hover → 无限循环

### 解决方案

```
container.onmousemove = () => {
  // 1. 找到鼠标最近的卡片
  // 2. 即时给它加 hover-light class（轻反馈）
  // 3. clearTimeout(hoverTimer) ← 每次鼠标动都重置
  // 4. 只有 closest !== activeIndex 时才启动新 timer
  // 5. timer 180ms 后触发整排平移
}
```

关键：监听放在**容器级别**而非单张卡片上。通过鼠标 X 坐标计算最近卡片槽位，不依赖卡片 DOM 的 mouseenter/mouseleave。卡片本身怎么移动都不影响命中判断。

## 四、入场动画

切换 Tab 时，卡片从下方 80px 处、缩放 0.8、透明度 0 淡入到最终位置。每张卡片间隔 80ms 的 stagger 延迟，形成依次展开的效果。
